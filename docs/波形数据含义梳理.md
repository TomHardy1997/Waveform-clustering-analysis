## 1. 这类数据本质上是什么

每个 Excel 文件可以理解为一个 cell 的 calcium imaging 时间序列。

原始表格通常有四列：

| 原始列名 | 含义 |
|---|---|
| `Time - F/F0` | F/F0 曲线对应的时间 |
| `F/F0 - F/F0` | 归一化荧光强度 F/F0 |
| `Time - dF/F0` | dF/F0 曲线对应的时间 |
| `F/F0 - dF/F0` | 相对基线变化量 dF/F0 |

直观理解：

- `F` 是当前时刻的荧光强度。
- `F0` 是基线荧光强度。
- `F/F0` 表示当前信号相对基线的倍数。
- `dF/F0` 通常表示 `(F - F0) / F0`，即相对基线的变化量。

如果 `F/F0 = 1.01`，大致表示比基线高 1%。  
如果 `dF/F0 = 0.01`，也大致表示相对基线升高 1%。

## 2. 单文件处理结果里四个 sheet 的含义

以 `处理结果_113520-1.xlsx` 为例，里面有四个 sheet：

### 2.1 原始数据

这个 sheet 是从原始 Excel 中提取并改名后的数据。

| 列名 | 含义 |
|---|---|
| `Time_F_F0` | F/F0 时间轴 |
| `F_F0` | 原始 F/F0 |
| `Time_dF_F0` | dF/F0 时间轴 |
| `dF_F0` | 原始 dF/F0 |

这一步还没有做平滑、基线校正、峰检测，只是把原始数据整理成统一列名。

### 2.2 预处理后数据

这个 sheet 是后续特征计算真正依赖的数据。

| 列名 | 含义 | 用途 |
|---|---|---|
| `时间(秒)` | 时间轴 | 横坐标 |
| `原始F/F0` | 原始 F/F0 | 用于保留原始信号 |
| `异常值截断后F/F0` | 3 倍标准差截断后的 F/F0 | 减少极端噪声影响 |
| `基线校正后F/F0` | 用前 10% 数据均值作为基线后校正 | 让曲线基线接近 1 |
| `平滑去噪后F/F0` | Savitzky-Golay 平滑后的 F/F0 | 用于找峰和统计 |
| `原始dF/F0` | 原始 dF/F0 | 反映相对基线变化 |
| `平滑去噪后dF/F0` | 平滑后的 dF/F0 | 用于观察变化趋势 |

当前脚本的预处理顺序是：

1. 对 `F/F0` 做 3 倍标准差异常值截断。
2. 取前 10% 时间点的均值作为 `F0_baseline`。
3. 用 `F/F0_clip / F0_baseline` 做基线校正。
4. 用 Savitzky-Golay 滤波器平滑。

### 2.3 峰值特征

这个 sheet 记录每一个被检测到的 calcium peak。

| 指标 | 含义 |
|---|---|
| `峰编号` | 当前 cell 中第几个峰 |
| `峰值时间(秒)` | 峰出现的时间 |
| `峰值幅度(ΔF/F0)` | 峰顶相对基线 1 的高度，即 `峰值 F/F0 - 1` |
| `半高宽(秒)` | 峰在半峰高度处的宽度，反映峰持续时间 |
| `半高宽左边界(秒)` | 半高宽左侧位置 |
| `半高宽右边界(秒)` | 半高宽右侧位置 |
| `10%-90%上升时间(秒)` | 从低水平上升到峰附近所需时间 |
| `平均上升速率(ΔF/F0/秒)` | 峰幅度 / 上升时间 |
| `90%-10%下降时间(秒)` | 从峰下降回低水平所需时间 |
| `平均下降速率(ΔF/F0/秒)` | 峰幅度 / 下降时间 |

这些指标可以粗略分成三类：

- 活动强度：峰值幅度、最大峰幅度。
- 活动频率：峰总数、发放频率。
- 动力学形态：半高宽、上升时间、下降时间、上升/下降速率。

### 2.4 整体统计特征

这个 sheet 把一个 cell 的所有峰和全时段信号汇总成一行指标。

以 `113520-1.xlsx` 为例：

| 指标 | 结果 |
|---|---:|
| 总时长 | 159.698 s |
| 采样频率 | 30 Hz |
| 总数据点数 | 4792 |
| F/F0 基线值 | 0.992395 |
| F/F0 全时段均值 | 1.000531 |
| F/F0 全时段标准差 | 0.003279 |
| F/F0 最大值 | 1.010904 |
| F/F0 最小值 | 0.995368 |
| 检测到的钙峰总数 | 7 |
| 钙峰平均幅度 | 0.003369 |
| 最大钙峰出现时间 | 40.366 s |

这些整体指标会进入批量汇总表 `批量处理汇总总表.xlsx`。

## 3. 时间段应该怎么理解

### 3.1 参考文献中的时间段

Li et al. 2022 的 in vivo RGC Ca2+ imaging protocol 是一个很清楚的刺激范式：

- 记录总长 60 s。
- 前 10 s 是刺激前 baseline，用来估计 `F0`。
- 中间 20 s 是 UV light stimulation。
- 后 30 s 是刺激结束后的恢复/offset response 观察期。

论文中用每个 RGC 的荧光信号 `F` 和刺激前基线 `F0` 计算 `ΔF/F0`，并根据光刺激 onset、offset 附近的波形变化，把细胞分为 ON、OFF、ON-OFF、sustained、transient 等活动类型。

### 3.2 你当前 `113520-1.xlsx` 的时间段

你的 `113520-1.xlsx` 总长约 159.7 s，和论文的 60 s 方案不完全一样。

从数据本身看，`113520-1` 的平滑 F/F0 有两个明显阶段转换：

- 约 39-40 s：F/F0 从基线附近快速升高，最大峰出现在 40.366 s。
- 约 80 s：F/F0 从较高状态回落到较低水平。

因此，当前这类 160 s 数据更合理的初步解释是：

| 时间段 | 初步含义 | 证据 |
|---|---|---|
| 0-40 s | 刺激前基线 / 适应期 | F/F0 维持在 1 附近 |
| 40-80 s | 刺激期或主要响应期 | 40 s 左右出现快速上升，之后维持较高水平 |
| 80-160 s | 刺激后恢复期 / offset 后阶段 | 80 s 左右明显回落，之后缓慢恢复或维持低水平 |

重要提醒：这个 40/80 s 分段是从曲线形状和峰值时间反推出来的。原始脚本没有记录“刺激开始时间”和“刺激结束时间”这两个参数。如果实验记录里有明确刺激 protocol，应该以后把它写进代码参数，比如：

```python
stim_on_time = 40
stim_off_time = 80
```

这样后续就可以分别计算 baseline、stimulus、post-stimulus 三段的均值、峰值、面积、持续响应等指标。

## 4. 每个特征值的实际意义

### 4.1 F/F0 基线值

脚本用前 10% 数据计算基线。

意义：

- 代表该 cell 在记录初期的平均荧光水平。
- 后续 F/F0 会除以这个值，使不同 cell 更可比。

注意：

- 如果前 10% 数据已经包含刺激或明显漂移，基线会被污染。
- 对当前 160 s 数据，前 10% 大约是前 16 s；如果真实刺激从 40 s 开始，则这个基线选择是合理的。

### 4.2 F/F0 全时段均值

代表整个记录期间信号的平均水平。

意义：

- 接近 1：整体接近基线。
- 大于 1：整体偏激活。
- 小于 1：整体偏抑制或信号下降。

但它不区分刺激前、中、后，所以解释能力有限。更推荐以后分段计算：

- baseline mean
- stimulation mean
- post-stim mean

### 4.3 F/F0 全时段标准差

代表整个记录期间信号波动程度。

意义：

- 标准差大：该 cell 活动变化大，可能响应强，也可能噪声大。
- 标准差小：信号稳定，可能响应弱或无响应。

注意：标准差本身不能区分“真实响应”和“噪声”，需要结合波形图和峰值特征。

### 4.4 F/F0 最大值和最小值

代表该 cell 在整个记录期间的最高/最低平滑信号。

意义：

- 最大值可近似代表 strongest activation。
- 最小值可提示 suppression 或 post-stimulus drop。

在 `113520-1` 中，最大值 1.010904，对应最大峰幅度 0.010904，也就是约 1.09% 的相对升高。

### 4.5 dF/F0 均值和标准差

dF/F0 更直接表示相对基线变化。

意义：

- dF/F0 > 0：相对基线上升。
- dF/F0 < 0：相对基线下降。
- 标准差越大，波动越强。

当前脚本主要用 F/F0 找峰，dF/F0 更多用于辅助观察。

### 4.6 检测到的钙峰总数

代表在平滑 F/F0 曲线上被 `find_peaks` 识别出来的峰数量。

意义：

- 峰数多：活动事件多，可能更活跃。
- 峰数少：活动事件少，可能响应单一或弱。

注意：当前脚本设置是：

```python
peak_height = 0.005
peak_prominence = 0.002
distance = sample_rate * 0.5
```

也就是说，至少间隔 0.5 s 才算不同峰。阈值会强烈影响峰数。

### 4.7 钙峰平均幅度

每个峰的 `peak F/F0 - 1` 后取平均。

意义：

- 反映平均响应强度。
- 对应论文中常说的 Ca2+ response amplitude。

如果峰幅度是 0.01，约等于 1% ΔF/F0。

### 4.8 半高宽 FWHM

峰在半峰高度处持续的时间。

意义：

- 半高宽短：transient，短暂响应。
- 半高宽长：sustained，持续响应。

这和论文中 ON-Transient、ON-Sustained、OFF-Sustained 这种分类思想是对应的。

### 4.9 上升时间和下降时间

上升时间：从低水平到峰附近的时间。  
下降时间：从峰回落到低水平的时间。

意义：

- 上升快：响应启动快，可能接近 transient onset response。
- 下降慢：响应持续，可能接近 sustained response。
- offset 后出现峰：可能类似 OFF response。

## 5. 论文和你当前数据的对应关系

### 5.1 可以直接借鉴的部分

论文给了一个重要思路：不要只看一个峰值，而是看整条波形在刺激 onset 和 offset 附近的动态。

论文中有这些功能响应类型：

| 论文类型 | 波形含义 |
---|---|
| ON-Sustained | 刺激开始后升高，并在刺激期间持续 |
| ON-Transient | 刺激开始后短暂升高，很快回落 |
| ON-OFF | 刺激开始和结束时都出现响应 |
| OFF-Transient | 刺激结束时短暂升高 |
| OFF-Sustained | 刺激结束后持续升高 |
| NR | 无明显响应 |

你当前做的 6 类波形聚类，本质上就是在学习类似的“波形形态类型”，只是当前命名更粗：

- 早峰
- 中峰
- 晚峰
- 后期下降型
- 回到基线/平稳型

### 5.2 不能直接照搬的部分

论文的 protocol 是 60 s：10 s baseline、20 s stimulation、30 s post-stimulus。

你的 `113520-1` 是 159.7 s，明显不是同一时间尺度。因此不能机械地把论文的 10/20/30 秒套到你的数据上。

更合理的做法是：

1. 根据实验记录确定真实 `stim_on_time` 和 `stim_off_time`。
2. 如果没有记录，则先用波形突变点近似估计，例如 40 s 和 80 s。
3. 分段计算特征。

## 6. 当前脚本的优点和需要注意的地方

### 优点

1. 能批量处理大量 cell。
2. 每个 cell 都有原始、预处理、峰值、整体统计。
3. 特征表适合后续聚类和比较。
4. 半高宽、上升/下降时间能反映动力学差异。

### 需要注意

1. 刺激时间没有显式写入脚本。
2. 基线默认前 10%，如果实验 protocol 改变，需要调整。
3. 峰检测阈值会影响峰数量和平均峰幅度。
4. 有些短文件或异常文件数值范围很大，可能需要单独质控。
5. 当前聚类脚本使用的是统计特征聚类；后面我们做的 6 类波形聚类则是按整条波形形状聚类，两者回答的问题不同。

## 7. 建议下一步怎么往下做

建议下一步把分析从“全时段统计”升级为“分段统计”。

如果确认刺激大约是 40-80 s，可以为每个 cell 增加：

| 新指标 | 含义 |
|---|---|
| baseline_mean | 0-40 s 平均值 |
| stim_mean | 40-80 s 平均值 |
| post_mean | 80 s 后平均值 |
| stim_delta | stim_mean - baseline_mean |
| post_delta | post_mean - baseline_mean |
| onset_peak | 40 s 后短窗口最大响应 |
| offset_peak | 80 s 后短窗口最大响应 |
| response_auc | 刺激期曲线面积 |
| recovery_slope | 刺激后恢复斜率 |

这样就可以更接近论文中的功能分类逻辑：

- ON response：刺激开始后升高。
- OFF response：刺激结束后升高。
- sustained response：刺激期间持续高于基线。
- transient response：短暂升高后快速回落。
- suppressed response：刺激后低于基线。
- no response：变化幅度落在噪声范围内。

## 8. 当前 `113520-1` 的一句话解释

`113520-1.xlsx` 是一个 159.7 秒、30 Hz 采样的单细胞 Ca2+ 波形。它在 0-40 s 基线附近较稳定，约 40 s 出现明显快速升高，40-80 s 维持较高水平，约 80 s 后回落到较低水平并进入恢复/后刺激阶段。脚本检测到 7 个峰，最大峰出现在 40.366 s，最大幅度约 0.0109 ΔF/F0，平均半高宽约 4.823 s，说明它有一个较明显的刺激相关响应，并带有一定持续成分。
